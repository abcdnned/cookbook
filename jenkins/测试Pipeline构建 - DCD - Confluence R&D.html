测试Pipeline构建

Skip to end of metadata
  • Created by Tom Yang, last modified on Jan 12, 2019
Go to start of metadata

本文旨在介绍通过Bitbucket触发jenkins pipeline的配置方法。以下用DP测试的构建作为示例。

WorkFlow

本示例使用Bitbucket Server Webhook to Jenkins插件作为触发构建的基础。Bitbucket上的支持Git Post Receive的插件还有多个，其他例如Parameterized Build for Jenkins能触发带参的Job。

构建由Bitbucket上手动TriggerBuild 或者Commit TriggerBuild 开始，触发Jenkins上中转任务，然后依次执行DP测试pipeline和DCD pipeline。

使用中转任务的原因是直接触发Pipeline构建需要更高版本的Bitbucket Server。

配置Jenkins Job

创建一个Freestyle Jenkins Job，并配置Jenkins上的Git Plugin。填入仓库URL，注意将Branches Specifier设置为**。

Enable Pool SCM选项

增加一个Trigger parameterized build one other projects的Post-build Actoins，配置好要跑的jenkins job名称。

配置Bitbucket插件

进入DP项目下的Setting页面（需要项目Admin权限），进入Hooks

配置插件的Jenkins URL。

配置Jenkins Pipeline

创建Jenkins Pipeline，JenkinsFile运行方式为从SCM获取，具体配置可以参考下面两个任务。

http://jenkins-dcd-master.dev.netis.com.cn:8080/view/BUILD/job/DP4-BUILD-PIPELINE/

http://jenkins-dcd-master.dev.netis.com.cn:8080/view/BUILD/job/DCD4-BUILD-PIPELINE/

写JenkinsFile时有两点需要注意，一是使用build语法来串联Pipeline。

Snippet
?
build(
  job: 'tomtest2',
  parameters: [
    [
      $class: 'StringParameterValue',
      name: 'BRANCH',
      value: rlsBr,
    ],
  ],
)

二是是利用notifyBitbucket插件将运行状态报告给Bitbucket。

Snippet
?
try {
...
notifyBitbucket('SUCCESS')
} catch (err) {
  notifyBitbucket('FAILED')
} finnaly {
...
}
 
 
def notifyBitbucket(String state) {
 
    if('SUCCESS' == state || 'FAILED' == state) {
        currentBuild.result = state         // Set result of currentBuild !Important!
    }
notifyBitbucket commitSha1: '', considerUnstableAsSuccess: true, credentialsId: 'c261343c-62b0-4fcb-83b6-03f1b6facfcc', disableInprogressNotification: false, ignoreUnverifiedSSLPeer: true, includeBuildNumberInKey: false, prependParentProjectKey: false, projectKey: '', stashServerBaseUrl: 'https://git.netisdev.com'
 
}

DP Pipeline Branch和DCD Pipeline Branch的关联方式

对于跑指定dp分支和dcd 分支的需求，目前提供两种关联方案：

  1. 根据DP ISSUE号码关联，例如DP分支和DCD分支名称中都有DP-1234，那么这两个任务就会被串联执行。
  2. 使用Pipeline的input功能，等待用户输入DCD分支名称。
Snippet
?
    userInput = input(
id: 'Proceed1', message: '申请输入：', parameters: [
[$class: 'TextParameterDefinition', defaultValue: '4.1/develop', description: '输入关联的DCD分支名', name: 'DCD BRANCH']
])
echo (userInput)

请求input时Pipeline会Block，对此可以用timeout设置超时时间（可行性未验证），并且发送notify给用户（hipchat，邮件，或者notifyBitbucket）
